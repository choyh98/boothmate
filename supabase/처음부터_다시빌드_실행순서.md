# Supabase 처음부터 다시 빌드 실행 순서

Supabase SQL Editor에서 아래 순서대로 실행하세요.

1. `00_전체삭제_초기화.sql`
   - 기존 public 스키마와 데이터를 전부 삭제합니다.
   - 새로 시작할 때만 실행합니다.

2. `01_전체초기설정_스키마_RLS_트리거.sql`
   - enum, 테이블, 인덱스, RLS 정책, 회원가입 트리거, 견적 제출/선택 함수, 관리자 뷰를 만듭니다.

3. `02_전시회_초기데이터_seed.sql`
   - 전시회 초기 데이터를 넣습니다.

4. 사이트에서 회원가입
   - 참여기업 계정 1개
   - 전시업체 계정 1개

5. `03_테스트계정_권한확인_trial설정.sql`
   - 회원가입 트리거 누락 계정을 보정합니다.
   - 전시업체를 `trial`, `approved`로 맞춰 견적 작성/제출 테스트가 가능하게 합니다.

6. Vercel 재배포
   - 로컬 코드 변경까지 배포해야 배포 사이트에서 동일하게 동작합니다.

확인할 흐름:
- 참여기업 로그인
- 전시 일정에서 견적 요청 작성
- 견적 요청 최종 제출
- 전시업체 로그인
- 공개 요청 상세 진입
- 견적 작성/임시저장/최종 제출
- 참여기업에서 받은 견적 확인/비교/선택
